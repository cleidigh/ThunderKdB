URL Link

사용법

웹페이지, 입력 폼, 혹은 전자 메일의 글을 선택해 오른쪽 클릭합니다.

  • URL로 보이는 글이라면, '선택한 URL 열기'를 통해 직접(현재/다른 탭으로) 열 수가 있습니다.
  • 링크가 보이지 않는 텍스트인 경우, 부메뉴가 표시됩니다. 강제로 링크처럼 취급할 수도 있고, 먼저 사용자에 맞게 변환할 수도 있습니다 (예：“www. ”과“. com”를 추가).
  • 환경설정 밑에서, 보여지는 부메뉴를 사용자 정의할 수 있습니다, 하이픈 만으로 이루어진 항목(예: "--")으로 당신만의 분리자를 추가할 수 있습니다.
  • 사용자 정의 메뉴 항목은 첫번째'*'로 선택한 것을 대체해 아무리 길더라도 모든 URL을 포함할 할수 있습니다. 짧은 항목을 '|'막대 앞에 넣어 메뉴 내 에서 보여질 별명을 가지게 할 수 있습니다: "단축명|http://www.원래이름.com/q=*".
  • Firefox의 경우, <Shift> 키를 누르면서 '열기'를 선택하면, 새 창으로 링크를 엽니다.
  • URL Link가 환경설정을 통해 표준 문맥메뉴의 위 또는 아래 주변에 설정을 위치시키는 것을 선택할 수 있습니다.

기능

  • URL Link는 Firefox와 Thunderbird용의 조그만 확장 기능으로, 메일/뉴스 메세지나 웹페이지의 비URL 텍스트를 선택해, 브라우저 화면에 열 수가 있습니다.
  • 전자 메일의 경우, 여러 줄로 나눠져 버린 링크도 이어 맞출 수 있습니다. 게다가 스페이스를 URL 문자 코드의%20로 바꿔놓아서, 네트워크 `file:' 이메일로 보내진 연결을 추적할 수 도 있습니다 (Windows의 X:나 UNC에 의한 네트워크 참조로부터 자동적으로 검출합니다).
  • 웹페이지의 경우, 웹페이지 또는 편집상자 내 문자 링크/URL를 선택을 허용, 실제의 링크일 경우 추적하게 됩니다. 언제나 링크를 추적할 수 있을 뿐만 아니라, mailto:링크도 분석합니다.
  • mailto:링크를 분석해 관련된 웹 사이트로 접속 할 수 있습니다.(예를 들면, mailto:someone@somesite.com을 http://www.somesite.com로 변환)
  • 다르게 팝업을 생성하는 javascript 연결을 '추적'할 수 있습니다(예：http://www.kelkoo.co.uk/ - 최근 이 사이트는 링크를 변경한 것 같아, URL Link가 효과가 없습니다만 (-_-^) …2.0이후 작업을 할 수 있길 원합니다.).
