ImportExportTools NG Descripción general y menús

ImportExportTools NG es una extensión de seguimiento basada en el trabajo original de Paolo "Kaosmos". La extensión ofrece múltiples funciones de importación y exportación que admiten varios formatos de archivo diferentes, entre los que se incluyen: Nota: algunos formatos no son compatibles con todas las funciones.
- mbox
- EML
- maildir
- HTML
- Texto sin formato
- CSV (valores separados por comas)
- PDF

Si bien la mayor parte de la funcionalidad de gira en torno a los mensajes, también se admite la importación y exportación de perfiles de Thunderbird. Además, las exportaciones también pueden incluir un archivo de índice HTML o CSV. La copia de seguridad del perfil de Thunderbird es otra característica.

El acceso a la funcionalidad IETng se realiza a través de varios menús diferentes instalados por la extensión. Las funciones son sensibles al contexto. Se agregan menús para:
- El contexto del panel de carpetas (operación de carpetas)
- El contexto de la lista de mensajes (mensajes seleccionados)
- El menú Herramientas (operaciones de carpeta y perfil)

Hay muchas opciones a las que se puede acceder en la entrada agregada en "Herramientas / Opciones de complementos" para las opciones de extensión. Cada uno de los menús también tiene un elemento de menú Opciones para acceder al diálogo de opciones.
Las nuevas opciones se documentan aquí, todo el conjunto se documentará posteriormente.

Formato de fecha personalizado

La opción de fecha personalizada permite el control completo sobre el formato de fecha utilizado para ambos archivos en el índice. Tanto el formato predeterminado "Fecha" como el nuevo formato personalizado "Fecha (personalizado)" están disponibles en los menús desplegables de nombre de archivo. La fecha personalizada también se puede usar con el token $ {date_custom} usado en los nombres de archivo personalizados extendidos. El formato de fecha original (predeterminado) es:
AAAAMMDDHHmm: 202011060808

El formato de fecha personalizado utiliza secuencias de% que coinciden con la funcionalidad tipo Unix de strftime() . Se basa en un puerto JavaScript de strftime() , una función para formatear la fecha y la hora.
Puerto de strftime () por TH Doan (https://thdoan.github.io/strftime/)

Descripciones de la secuencia de conversión

%g Como %G , pero sin siglo, es decir, con un año de 2 dígitos (00-99).
%H Hora como número decimal utilizando un reloj de 24 horas (rango de 00 a 23). Consulte también %k .

%m Mes como número decimal (rango de 01 a 12).

%M Minuto como número decimal (rango de 00 a 59).
%p Ya sea "AM" o "PM" según el valor de tiempo dado. El mediodía se trata como "PM" y la medianoche como "AM".
%P Como %p pero en minúsculas ("am" o "pm").
%s Número de segundos desde la Época, 1970-01-01 00:00:00 +0000 (UTC).
%S Segundo como número decimal (rango de 00 a 59).

Notas de compatibilidad:

  • %c - la cadena formateada es ligeramente diferente
  • %D - no implementado (use %m/%d/%y o %d/%m/%y )
  • %e - no se agrega espacio
  • %E - no implementado
  • %h - no implementado (use %b )
  • %k - no se agrega espacio
  • %n - como %m , pero sin cero a la izquierda (use \n para nueva línea)
  • %O - no implementado
  • %r - no implementado (use %I:%M:%S %p )
    • ${subject} El asunto del mensaje
  • %R - no implementado (use %H:%M )
  • %t - no implementado (use \t )
  • %T - no implementado (use %H:%M:%S )
  • %U - no implementado
  • %W - no implementado
  • %+ - no implementado
  • %% - no implementado (use % )


Formato de nombre de archivo extendido:


El formato de nombre de archivo extendido utiliza tokens. El nombre de archivo exacto se puede construir utilizando los tokens, así como cualquier texto o caracteres de nombre de archivo válidos. Los caracteres no válidos son: | * : < > ? \ / " Los tokens en inglés (en-US) están disponibles en cualquier lugar. Los tokens específicos del lugar también están disponibles y se enumeran debajo de la tabla en inglés.

${subject}-${date}-${index} - El formato predeterminado
${prefix}_${date_custom}_${sender} : usa el formato de fecha personalizado en lugar del predeterminado
${subject}-GmailMsg-${date}-${index} - Con texto personalizado

Preferencias nuevas y experimentales (opciones)

Hay varias funciones y opciones nuevas que aún no se pueden configurar en el cuadro de diálogo de opciones. Una vez que estas funciones se estabilicen, se agregarán a la interfaz de usuario.
Para poder utilizarlos, deben estar configurados en el Editor de configuración. Para acceder al editor, use el menú Herramientas / Opciones, luego seleccione la sección Avanzado y habrá un botón 'Editor de configuración' en la parte inferior. Si escribe 'importexporttoolsng' en la barra de búsqueda, obtendrá todas las preferencias de extensión. A continuación, puede editar las distintas preferencias a continuación según sea necesario.

importexporttoolsng.export.index_date_custom_format

Esta preferencia sigue el mismo formato completo que se utilizará para los archivos de índice. Es una cadena vacía por defecto, si tiene una cadena de fecha / hora formateada, esto cambiará el formato de la fecha del índice.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Ambas preferencias usan el mismo tipo de formatos de nombres de archivos extendidos que usa el nombre de archivo de los mensajes exportados. El primero es para archivos adjuntos externos, el segundo es para archivos adjuntos integrados (en línea).

importexporttoolsng.experimental.use_delivery_date

Normalmente, la 'Fecha' de un mensaje en Thunderbird es la fecha establecida por el cliente de correo del remitente. Establecer esta opción en "verdadero" utiliza la fecha de recepción en el cliente de correo del destinatario. Esta fecha se utilizará tanto para los nombres de archivo como para la fecha en los archivos de índice.

Función básica global de teclas de acceso rápido (accesos directos)

importexporttoolsng.experimental.hot_keys

Esta nueva preferencia permite al usuario configurar hasta 10 atajos de teclado. Los atajos pueden realizar la mayoría de las funciones de ImportExportTools NG, así como otras funciones de Thunderbird. La cadena de preferencias tiene el formato de un objeto con formato JSON. El formato y las claves son los siguientes:

Ejemplo de entradas de teclas de acceso rápido (matriz de objetos JSON)

id - el índice de la clave (1-10)
key - la clave (carácter único)
keycode clave - el 'VK_ *' define el código clave
Se puede incluir una clave o un código clave, pero no ambos
modifiers : los modificadores de tecla, lista separada por espacios ('control', 'shift', 'alt', 'accel')
oncommand : una cadena de comando directo
contexts : el contexts la ventana donde la clave está activa
'all' : todas las ventanas de Messenger, redactar, buscar
'messenger' : la ventana principal de mensajería y sus descendientes
'compose' - la ventana de redacción del mensaje
'search' - el diálogo de búsqueda
